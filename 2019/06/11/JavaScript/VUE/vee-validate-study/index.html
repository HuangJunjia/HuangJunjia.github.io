发表于 2019-06-11
因为最近工作内容中需要用到自定义表单验证，前端ui框架用的是mand-mobile，在预览doc的时候看到官方推荐使用vee-validate，这里记录一下vee-validate的使用……

前言

在最近的工作中，我的领导说需要设置自定义表单验证，因为ui框架用的是mand-mobile(以下简称md)，在md的官方文档中有说明可以使用第三方工具，如vee-validate，故记录一下使用记录。

因为项目是vue工程，所以接下来的内容都以vue为准，版本是2.5.2

一、如何安装和引入

  1. 开始

    1. 安装

    2. 引入

  2. 因为项目目前以简体中文为主，所以修改一下vee-validate的默认语言以及对应的报错信息语言包

    但是这里需要注意一点，默认的中文语言包估计是机翻的，翻译的有点跟常用的不同，或者因为业务需求有需要不同的错误提示文本，所以在此我们去vee-validate的github下载一下他们的语言包，然后我们自己拿来修改，具体路径为locale，如果需要，你可以全部下载。

    修改语言包之后引入

    这个时候我们就可以到页面中查看我们修改之后的提示文本了~

二、使用

官方文档中的引用需要编写一个name值对应的span（也可以是其他标签）去显示校验结果，因为在md中md-input-item中已经有一个属性error用于显示错误信息，所以我们就直接使用了

PS：这里很坑爹，因为需要通过name值去确定校验的表单元素，同时这个name值也是反显的错误信息的主要内容之一，所以这里的name值只好写成了中文了……
