将脚本添加到环境中

我们平常用npm去安装一个全局cli的时候用的是-g的命令，然后在电脑的任何地方都可以使用了，而且他们也不是通过./index.js这样的命令去执行，而是有一个指令，比如webpack-cli。

我们要实现这样的方式，可以有两种做法：

  1. 将该脚本添加到系统环境变量中。
  2. 通过npm链接到全局模块中。

第一种做法我就不详细解释了，请自行百度，这里我们着重讲第二种。

我们大家都知道，我们通过npm install -g安装的全局模块都可以直接使用对应的指令，那那些模块到底长啥样，我们可以找到npm全局仓库中找到他们：

yargs修改帮助信息

在平常使用的命令行中，我们可以使用-h或者--help参数打印帮助信息，yargs模块提供了4个方法生成帮助信息：

  1. usage：用法格式
  2. example：提供例子
  3. help：显示帮助信息
  4. epilog：出现在帮助信息的结尾
